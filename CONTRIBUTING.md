# Contributing to Garmin Data Analyzer

Thank you for your interest in contributing to this project!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/garmin-data-analysis.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit: `git commit -m "Add your feature"`
6. Push: `git push origin feature/your-feature-name`
7. Open a Pull Request

## Development Setup

```bash
# Install dependencies
pip install -r requirements.txt

# Create .env file with your Garmin credentials
cp .env.example .env
# Edit .env with your credentials
```

## Code Style

- Follow PEP 8 guidelines
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and concise

## Commit Messages

- Use clear, descriptive commit messages
- Start with a verb in present tense (Add, Fix, Update, etc.)
- Keep the first line under 50 characters
- Add details in the body if needed

## Testing

Before submitting a PR:
- Test your changes manually
- Ensure no errors in the output
- Verify GUI functionality if applicable

## Questions?

Feel free to open an issue for any questions or suggestions!
